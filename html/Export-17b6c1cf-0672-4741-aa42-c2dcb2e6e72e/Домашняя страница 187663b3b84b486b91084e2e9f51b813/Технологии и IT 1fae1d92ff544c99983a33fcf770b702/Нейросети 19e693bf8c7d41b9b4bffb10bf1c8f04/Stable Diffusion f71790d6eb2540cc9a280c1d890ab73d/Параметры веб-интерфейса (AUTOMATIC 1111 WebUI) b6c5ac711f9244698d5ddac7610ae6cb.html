Основные параметры txt2img (Текст в изображение)

✍ Promt & Negative Prompt (Запрос и отрицательный запрос)

  • Promt — текстовый запрос использующийся для генерации изображения.
  • Negative Promt — текстовый запрос убирающий перечисленные свойства при генерации изображения.

💻 Sampling Steps & Method (Шаги и метод выборки)

Sampling Steps (Шаги выборки)

Количество итераций (шагов) выполнения процесса выборки. Для генерации изображений содержащего подробный запрос с большим количеством деталей, рекомендуется использовать большее количество шагов, в ином случае это будет избыточно и модель не будет вносить изменений после некоторого количество шагов. Стоит отменить, что это зависит от используемой модели и некоторые показывают результаты лучше при большем количестве шагов. С увеличением значения повышается потребление VRAM и увеличивается время генерации. Большое значение не всегда гарантирует генерацию хорошего изображения. Значение по умолчанию (равное 20) дает хорошие результаты, но можно увеличивать его до максимального значения в 110.

Примеры использования различных значений:

  • 20 шагов — Неудачное значение шагов, т.к. лицо не прорисовалось.
  • 30 шагов — Хороший результат с соблюдением баланса деталей и пропорций, без видимых артефактов генерации.
  • 40 шагов — Большее количество деталей, но неудачные пропорции.
  • 50 шагов — Новый вариант композиции с интересным результатом (сгенерировался капюшон, хотя в первоначальном запросе он отсутствовал).

Sampling Method (Метод выборки, семплер)

Существует множество методов выборки, основными отличительными характеристиками которых является: количество шагов для получения приемлемого результата и уклон в определенный стиль. Семплеры не все дают разный результат и это то не нейросеть в нейросети, а метод решения квадратных уравнений использующийся для генерации изображения из шума. Большинство семплеров в интерфейсе дают примерно одинаковый результат, но при разном количестве шагов.

📏 Size (Ширина и высота)

Параметр указывающий на размер генерируемого изображения (соотношение сторон).

Примеры использования различных значений:

🌱 Seed (Ключ генерации)

Параметр является ключом генерации указывающим на набор случайных пикселей разных цветов для генерации шума, на основе которого будет генерироваться изображение.

⚖️ CFG Scale (Степень влияния запроса)

Параметр CFG Scale (Clip Guidance, Guidance Scale) указывает на сколько итоговое изображение будет походе на запрос. Чем больше значение, тем больше прислушивается к запросу, не внося дополнительных деталей. На русском звучит как “шкала безклассификаторного управления диффузией” 😮.

  • При низком значении параметра CFG Scale генерируются более креативные варианты изображения, т.к. нейросеть больше добавляет “от себя.
  • При высоком значении параметра CFG Scale генерируются изображения с ограниченной вариативностью (следует использовать при генерации изображений с точным запросам).

Примеры использования различных значений при 30 шагах выборки:

  • 4,0 CFG Scale — удовлетворительный результат.
  • 5,5 CFG Scale — хороший результат.
  • 12,5 CFG Scale — хороший результат с детализированным изображением.
  • 24,0 CFG Scale — плохой результат (следует уменьшить значение).

🛍 Batch Generation (Генерация пачками)

  1. Batch size (Количество изображений в пачке) — количество картинок, которое будет генерироваться одновременно. Слабо влияет на время генерации и зависит от объема VRAM.
  1. Batch count (Количество пачек изображений) — количество запуска генерации с одним и тем же запросом. Сильно влияет на время генерации.

Дополнительные параметры txt2img (Текст в изображение)

📐 Upscale (Увеличение изображения)

Увеличение исходного изображения с использованием другой нейросети.

Можно выбрать различные нейросети для апскейла и степень увеличения (x2, x4 и т.п.).

Для апскейла фото и архитектуры лучше скачать модель Remacri от foolhardy.

Для лучшего качества лучше использовать скрипт SD upscale на вкладке img2img, который нарезает исходную картинку на тайлы и на их основе делает “обрисовывание” (то же самое, что и Highres. fix по сути). Параметр шум (denoising strength) лучше держать в районе 0.2 для сохранения композиции. Такой метод позволяет исправить некоторые места и добавить детализации, которые в изначальном разрешении (512x512) отсутствуют. И только после этого переходить к Upscale увеличивая уже эти мелкие детали.

📷 Highres. fix (Исправление высокого разрешения)

Параметр исправляет проблему с изменением композиции, при увеличении размера изображения (Size) более 512x512.

При генерации изображений в большом разрешении часто происходит “задвоение” объектов и прочие артефакты, а при использовании этого параметра сначала генерируется уменьшенное изображение, а потом, сохраняя композицию, а потом генерируется изображение уже в увеличенном разрешении, используя маленькую картинку с наложенным на неё шумом. Но если значение параметра шум (denoising strength) будет более 0.2, то детали и даже композиция всё равно поменяются.
